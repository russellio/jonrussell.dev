# WARP.md

This file provides guidance to WARP (warp.dev) when working with code in this repository.

## Overview

This is a personal portfolio website built as a Laravel 11 + Vue 3 + Inertia.js monolith. The project showcases software engineering projects and experience with a modern tech stack, full TypeScript support, and Tailwind CSS 4 styling.

**Tech Stack:**
- **Backend:** Laravel 11, PHP 8.4, SQLite
- **Frontend:** Vue 3 (Composition API), TypeScript, Inertia.js
- **Styling:** Tailwind CSS 4, custom CSS variables
- **State:** Pinia stores
- **Build:** Vite with SSR support
- **Testing:** PEST (PHP), ESLint (JS/TS)

## Development Commands

### Setup
```bash
# Install dependencies
composer install
npm install

# Environment setup
cp .env.example .env
php artisan key:generate
php artisan migrate
```

### Development Server
```bash
# Start all development services (server, queue, logs, vite) - RECOMMENDED
composer dev

# OR start services individually
php artisan serve        # Laravel dev server (port 8000)
npm run dev              # Vite dev server with HMR

# With SSR
composer dev:ssr         # Build and run with server-side rendering
npm run build:ssr        # Build assets for SSR
```

### Testing & Linting
```bash
# PHP tests
./vendor/bin/pest                    # Run all tests
./vendor/bin/pest --filter=TestName  # Run specific test

# Code quality (PHP)
./vendor/bin/pint                    # Format PHP code (Laravel Pint)

# Frontend linting & formatting
npm run lint                         # Run ESLint with auto-fix
npm run format                       # Format with Prettier
npm run format:check                 # Check formatting without changes
```

### Building for Production
```bash
npm run build                        # Build frontend assets
npm run build:ssr                    # Build with SSR support
```

### Type Checking
```bash
npx vue-tsc --noEmit                 # TypeScript type checking for Vue files
```

## Architecture

### Routing & Navigation
- **Backend routes:** `routes/web.php` defines Inertia page routes
- **Wayfinder:** Laravel Wayfinder auto-generates type-safe route helpers in `resources/js/routes/index.ts`
- Routes are imported as functions: `import { home, about, projects } from '@/js/routes'`
- **Pages:** `resources/js/pages/*.vue` - Inertia page components (Projects.vue, About.vue)

### Frontend Structure
```
resources/js/
├── app.ts                    # Inertia app entry point
├── ssr.ts                    # SSR entry point
├── App.vue                   # Root component wrapper
├── pages/                    # Inertia page components
├── layout/                   # Header, Footer, Nav
├── components/               # Reusable UI components
│   ├── Modal.vue             # Base modal component
│   ├── BackgroundStars.vue   # Animated background
│   └── modals/               # Specific modal implementations
├── stores/                   # Pinia state stores
│   └── modalStore.ts         # Global modal state management
├── composables/              # Vue composables
│   └── useModal.ts           # Modal logic abstraction
├── data/                     # Static data files
│   └── projects.json         # Project portfolio data
├── types/                    # TypeScript type definitions
└── routes/                   # Auto-generated Wayfinder routes
```

### State Management
- **Pinia stores** for global state (see `resources/js/stores/`)
- **Modal system:** Central `modalStore.ts` tracks open modals by ID, `useModal.ts` composable provides reusable modal logic
- Stores are registered in `app.ts` via `createPinia()`

### Project Data
- Portfolio projects are defined in `resources/js/data/projects.json`
- Each project contains: id, title, description, skills, technologies, tools, company, images, and links
- Add new projects by editing this JSON file following the existing schema

### Styling
- **Tailwind CSS 4** configured via `tailwind.config.ts`
- Custom CSS variables in `resources/css/app.css`
- Font configuration uses `@fontsource` packages
- Responsive breakpoints: mobile (<768px), tablet (768-1024px), desktop (>1024px)

### Path Aliases
- `@/` resolves to `resources/` directory (configured in `vite.config.ts` and `tsconfig.json`)
- Use `@/js/`, `@/css/` for imports

### Key Laravel Conventions
- Controllers in `app/Http/Controllers/`
- Models in `app/Models/`
- Database migrations in `database/migrations/`
- SQLite database at `database/database.sqlite`

## Important Notes

### Inertia.js SPA Architecture
- This is NOT a traditional multi-page Laravel app
- Pages are Vue components rendered via Inertia
- No Blade views for pages (only `resources/views/app.blade.php` as shell)
- Navigation uses Inertia's SPA routing, not full page reloads

### Auto-Generated Files
- `resources/js/routes/` is auto-generated by Wayfinder - DO NOT edit manually
- `auto-imports.d.ts` and `components.d.ts` are auto-generated type definitions

### Development Workflow
- Use `composer dev` for integrated development (server + queue + logs + vite)
- Run tests before committing changes
- Format code with Prettier and Pint before pushing
- For route changes: edit `routes/web.php`, Wayfinder will regenerate `resources/js/routes/index.ts`

### Testing
- PHP tests use PEST framework (`tests/Feature/`, `tests/Unit/`)
- Test database uses in-memory SQLite (configured in `phpunit.xml`)
- No frontend test suite currently configured
